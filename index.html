Navigation & Structure

Three layers: the dark sidebar for navigation, a light grey background, and white cards for content.

Dashboard — Where you see these colors
  • Dark sidebar (#0F172A) — The left panel. Always visible. This is how you move between Party Master, Billing, Payments, etc. The active page has a blue bar next to it.
  • Light grey background (#F1F5F9) — The area behind all content. Subtle, easy on the eyes for long working hours.
  • White cards (#FFFFFF) — Forms, tables, and data sit on white surfaces. White = "this is where the work happens."
  • Thin borders (#E2E8F0) — Separate sections without being distracting. You barely notice them, but they keep everything organized.

Actions & Selection

Blue is the action color. Whenever you see blue, it means "click me" or "this is active right now."

Party Master — Where you see blue
  • Blue sidebar indicator — The blue bar on the left shows which page you're on (Party Master in this case).
  • Blue "New Party" button — Primary action buttons are always blue. This is the main thing you can do on this page.
  • Blue focus ring — When you click into a text field (like the search box), a blue glow appears around it so you know where your cursor is.
  • Grey secondary buttons — Less important actions like "Cancel" are grey, so they don't compete with the main blue action.

Forms & Text Hierarchy

Not all text is equally important. Darker = more important. Lighter = supporting detail.

Party Form — How text colors guide your eye
  • Dark text (#0F172A) — Field values you type in (party name, address). The most important information.
  • Medium grey (#475569) — Labels like "Party Name", "GSTIN", "Address". Tells you what each field is.
  • Light grey (#64748B) — Placeholder text and hints. Disappears when you start typing.
  • Blue "Save Party (F3)" button — The primary action. Disabled (greyed out) until you fill in required fields.

Components & Layout

The building blocks of every screen — buttons, inputs, badges, and the compact layout designed for speed.

Compact Density

Designed for speed, not beauty. Empire users enter 50+ bills per day. Every row is 32px tall — the same density as Empire. We're not building a consumer app with lots of whitespace. We're building a precision instrument.

Form Fields

Every input field is 32px tall — same as table rows. This keeps the form compact so clerks can see all fields at once without scrolling.

Keyboard Shortcuts (F-Key Bar)

Just like Empire, ThreadBooks uses F-keys for common actions. The dark bar at the bottom of every screen shows which shortcuts are available.

Migration Wizard

New feature — automatically imports data from Empire's Access/SQL Server databases.

Data Import Flow
Empire (Current)
New Feature — No Empire Equivalent

Empire data lives in Access (.mdb) and SQL Server databases. Moving between computers or to a new system requires manual database export/import — a process most clerks can't do without IT support.

ThreadBooks (New)
  • Step-by-step wizard guides users through importing their Empire data
  • Automatic field mapping — party names, GST numbers, balances, all carried over
  • Runs on first launch, so existing Empire users can switch with zero data loss

Money: Debit & Credit

The most important colors in an accounting app. Red = money owed (debit). Green = money received (credit). You'll see these everywhere — in badges, ledger entries, and outstanding balances.

Geist Mono (Monospace) — Only used for money and tax numbers. Every digit is the same width, so ₹45,678.00 and ₹1,23,456.00 line up perfectly in columns. Indian number formatting (1,23,456) is used throughout. Red = debit, green = credit.

Components & Layout

The building blocks of every screen — buttons, inputs, badges, and the compact layout designed for speed.

Compact Density

Designed for speed, not beauty. Empire users enter 50+ bills per day. Every row is 32px tall — the same density as Empire. We're not building a consumer app with lots of whitespace. We're building a precision instrument.

Form Fields

Every input field is 32px tall — same as table rows. This keeps the form compact so clerks can see all fields at once without scrolling.

Keyboard Shortcuts (F-Key Bar)

Just like Empire, ThreadBooks uses F-keys for common actions. The dark bar at the bottom of every screen shows which shortcuts are available.

Migration Wizard

New feature — automatically imports data from Empire's Access/SQL Server databases.

Data Import Flow
Empire (Current)
New Feature — No Empire Equivalent

Empire data lives in Access (.mdb) and SQL Server databases. Moving between computers or to a new system requires manual database export/import — a process most clerks can't do without IT support.

ThreadBooks (New)
  • Step-by-step wizard guides users through importing their Empire data
  • Automatic field mapping — party names, GST numbers, balances, all carried over
  • Runs on first launch, so existing Empire users can switch with zero data loss
